SOC Triage Assistant — Architecture

read-only LLM triage · strict JSON schema · kill switch at every step

Pipeline

  • • Alert arrives (Wazuh / Elastic / syslog / JSONL)
  • • Kill switch checked — skip if armed
  • • Prompt built with few-shot examples
  • • LLM called, response validated against schema
  • • Record persisted, confidence threshold applied
  • • Render Markdown / Slack / Teams on demand

Data model

  • • triage_records: alert → triage result
  • • feedback: 👍/👎 verdicts with comment + operator
  • • prompt_bank: few-shot examples auto-grown from 👍
  • • SQLite WAL for analytics, JSONL for forensics
  • • kill_switch_audit.jsonl: every state transition

Six safety properties

  • • 1. Strict JSON schema — never trust free-form LLM output
  • • 2. Kill switch at every alert — even mid-batch
  • • 3. Read-only against SIEM — GET endpoints only
  • • 4. No action execution — recommendations are text only
  • • 5. Confidence < 0.6 → needs_human_review
  • • 6. Full kill-switch audit log
